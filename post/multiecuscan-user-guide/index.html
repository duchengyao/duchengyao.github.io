Multiecuscan (MES) 中文用户手册

发表于 2023-04-25 分类于 赛车 阅读次数：

Multiecuscan 是意大利车辆的诊断软件。 它允许您对支持的车辆/模块执行各种诊断任务。

为了使用该软件，您需要一个 OBD 接口。 软件目前支持以下接口：KL（也称为 VagCom 409）、ELM327（1.3或更新版本）、OBDKey 1.40、OBDLink、ELM Scan 5、CANtieCAR。 也支持蓝牙 OBDKey、ELM 327 和 OBDLink 接口，但不推荐用于特殊功能（如 PROXI 对准、远程控制编程、IMA 编码等）！

CANtieCAR 是具有多路复用功能的专业多协议接口。

Multiecuscan 目前有三个版本——免费版、注册版和多路复用版。

1. 安装和许可

1.1. Multiecucan 安装

Multiecuscan 是一个 Microsoft .NET 应用程序。 它需要在您的计算机上安装 Microsoft .NET Framework 才能使用它。 Microsoft .NET Framework 是免费的，可以从 www.microsoft.com 下载。 Multiecuscan 至少需要 Microsoft .NET Framework 2.0 SP1（3.5 版也可以完美运行）。

请注意，如果您使用的是 Windows Vista 或 Windows 7，那么您的计算机上很可能已经安装了 Microsoft .NET Framework。

Multiecuscan 适用于以下操作系统：Windows XP、Windows Vista 和 Windows 7。它同时支持 32 位和 64 位版本的 Windows。

为了在您的计算机上安装 Multiecuscan，您需要从 www.multiecuscan.net 下载安装程序。 然后您只需运行安装文件并按照屏幕上的说明进行操作。

1.2. 接口

为了使用 Multiecuscan 软件，您需要有一个接口将您的计算机连接到车辆的 OBD 端口。

请注意，所有接口都有一个 COM 端口号。 它不仅适用于旧的串行端口接口。 USB 和蓝牙接口也有一个 COM 端口号，因为它们被 Windows 识别为串口。 如果您没有在 Windows 的设备管理器中看到该接口，那么您可能没有正确安装它的驱动程序。

Multiecuscan 经过测试并支持以下接口：

KL / VagCom 409 接口

这是最基本的接口类型。 这些接口主要适用于旧车。 通过这种类型的接口，您可以连接到大多数发动机控制模块、大多数变速箱控制模块、旧式 ABS 模块、旧式动力转向模块、旧式安全气囊模块和大多数 XENON 控制模块。

这些接口不支持连接到车辆的 CAN 网络。 为了连接到 ABS、安全气囊、动力转向和 XENON 模块，您需要一根适配器电缆，或者您需要修改您的接口。

该接口可以通过 USB 或串行连接到计算机。 在这两种情况下，它都被计算机识别为串口！

注意：您可以查看以下链接以了解您的汽车需要哪种类型的适配器/改装：

http://www.multiecuscan.net/SupportedVehiclesList.aspx

安装说明：

首先需要安装接口的串口驱动。 驱动程序通常随您的接口一起提供。 如果您没有驱动程序，那么您可以尝试从 FTDI 站点下载并安装驱动程序。

http://www.ftdichip.com/Drivers/CDM/CDM20814_Setup.exe

成功安装驱动程序后，您必须更改已安装串口的默认配置才能使其正常工作。 转到 Windows 的设备管理器并双击 USB 串行端口（或右键单击并选择属性）。

转到端口设置并单击高级

在下一个屏幕上，为“延迟计时器”选择可能的最低值

如果您有内置串行端口或 PL2303 USB 串行端口，您需要将缓冲区降低到最小值，如下一个屏幕截图所示。 请注意，对于 PL2303 USB 串行端口，您需要修改一些注册表设置以使其支持 10400 bps。

现在，更改这些设置后，您必须在 Multiecuscan 的设置窗口中将接口类型设置为 K-Line/VAGCOM。 您还必须设置正确的串行端口。

ELM 327 接口

这是一种更高级的接口类型，支持多种协议。 连接到车辆的 CAN 网络需要 ELM 327 版本 1.3 和更新版本。

请注意，中文 ELM 327 接口上的版本标记与正版 ELM 接口不同。

使用这种类型的接口，您可以连接到所有支持的模块，除了大多数旧的安全气囊模块和一些旧的 ABS 和引擎模块。

为了连接到 ABS、安全气囊、动力转向和 XENON 模块，您需要一根适配器电缆，或者您需要修改您的接口。

此外，为了连接到 Grande Punto、Punto Evo、500、Mito 和一些 Doblos 中的大多数控制模块，您需要一根适配器电缆。

注意：您可以查看以下链接以了解您的汽车需要哪种类型的适配器/改装：

http://www.Multiecuscan.net/SupportedVehiclesList.aspx

ELM 327 接口警告：请注意几乎所有 最近几个月生产的中文ELM 327接口访问CAN网络有问题。 只需从接口上移除一个电阻器即可轻松解决此问题。 该过程包含在论坛的以下主题中：

http://forum.multiecuscan.net/viewtopic.php?f=5&t=1493

安装说明：

首先需要安装接口的串口驱动。 驱动程序通常随您的接口一起提供。 如果您没有驱动程序，那么您可以尝试从 FTDI 站点下载并安装驱动程序。

成功安装驱动程序后，您必须在 Multiecuscan 设置中将接口类型设置为 ELM 327 v1.3+。 您还必须设置正确的串行端口和端口速度。 所有 ELM 327 v1.3 接口的端口速度通常为 38400。 您可以使用测试按钮来测试您的 ELM 327 接口。 它显示了 ELM 327 软件的版本，还测试了 ELM 接口如何响应各种命令。 您应该在每个命令下看到一个 OK 响应。

如果您看到一条消息说 COMX 不存在，则表示您选择了不正确的串行端口。 如果您看到一条消息说超时，这很可能意味着您选择了不正确的端口速度

OBDKey接口

这也是一种支持多种协议的高级接口类型。 连接到 Fiat、Alfa 和 Lancia 的 CAN 网络需要 OBDKey 1.4 及更新版本。

使用这种类型的接口，您可以连接到所有支持的模块。

为了连接到 ABS、安全气囊、动力转向和 XENON 模块，您需要一根适配器电缆，或者您需要修改您的接口。

此外，为了连接到 Grande Punto、Punto Evo、500、Mito 和一些 Doblos 中的大多数控制模块，您需要一根适配器电缆。

OBDLink 接口

这也是一种支持多种协议的高级接口类型。 使用这种类型的接口，您可以连接到所有支持的模块，除了一些旧的安全气囊模块和一些旧的 ABS 和引擎模块。

为了连接到 ABS、安全气囊、动力转向和 XENON 模块，您需要一根适配器电缆，或者您需要修改您的接口。

此外，为了连接到 Grande Punto、Punto Evo、500、Mito 和一些 Doblos 中的大多数控制模块，您需要一根适配器电缆。

ElmScan 5 接口

这也是一种支持多种协议的高级接口类型。 在 Multiecuscan 中，您必须选择“ELM 327 1.3+”和端口速度 115,200 bps。 使用这种类型的接口，您可以连接到所有支持的模块，除了大多数旧的安全气囊模块和一些旧的 ABS 和引擎模块。

为了连接到 ABS、安全气囊、动力转向和 XENON 模块，您需要一根适配器电缆，或者您需要修改您的接口。

此外，为了连接到 Grande Punto、Punto Evo、500、Mito 和一些 Doblos 中的大多数控制模块，您需要一根适配器电缆。

蓝牙接口

Multiecuscan 支持蓝牙 ELM、OBDKey 和 OBDLink 接口。 但是，由于这些接口的一些限制，不建议将它们用于特殊功能（如 PROXI 对准、远程控制编程、IMA 编码等）！

CANtieCAR接口

这是一个具有多路复用能力的专业多功能多协议接口。使用此接口，您可以连接到所有支持的模块。 您不需要额外的适配器电缆或修改即可连接到所有模块（旧车辆所需的 3 针适配器除外）。

此接口可以在 USB 或蓝牙模式下工作。 蓝牙模式完全支持所有模块。 请注意，如果您想在 USB 模式下使用接口，则必须先将其连接到计算机，然后再连接到车辆。

此接口包含在 Multiecuscan MULTIPLEXED 捆绑包中。

安装说明：

首先需要安装接口的驱动程序。 为了安装驱动程序，您需要将 CANtieCAR 接口连接到计算机上的 USB 端口。 请注意，接口不应连接到汽车，它只能连接到 USB 端口！

然后 Windows 将开始搜索驱动程序。 它实际上会检测到 2 个不同的设备——一个 USB 驱动器和一个串行端口接口。 USB 驱动器应该会自动安装，因为它使用的是标准 Windows 驱动程序。 串行接口的驱动程序位于 USB 驱动器上。 因此，一旦系统询问您串行设备驱动程序的位置，您必须将其指向 USB 驱动器。

Windows 可能会警告您驱动程序未签名，只需选择“无论如何安装此设备驱动程序”选项即可。 然后 Windows 将安装驱动程序，如下面的屏幕截图所示。

CANtieCAR 接口现已安装。

在 USB 驱动器上，您可以找到用于 CANtieCAR 接口配置和固件升级的附加软件。

Multiecuscan MULTIPLEXED 的安装文件也位于 USB 驱动器上。

1.3. Multiecuscan 许可证

为了使用 Multiecuscan 的全部功能，您需要购买许可证。 许可证以许可证密钥的形式出现，您必须在 Multiecuscan 的注册窗口中输入该密钥。

首先，您必须下载并安装 Multiecuscan。

然后你必须启动程序并打开注册窗口。 要打开注册窗口，您必须单击蓝色的注册按钮，如下面的屏幕截图所示。

在注册屏幕上，您可以看到计算机的硬件密钥。 您将需要此密钥才能购买许可证。

注意：硬件密钥对于 Multiecuscan MULTIPLEXED 捆绑包的所有者来说并不重要。

有了您的许可证，您将收到一个许可证密钥，您必须在此处输入该密钥，然后单击激活按钮。 如果许可证密钥正确，程序将关闭。 然后你必须重新启动它，它就会被激活。

注意：如果您拥有 Multiecuscan MULTIPLEXED 许可证，则在激活期间必须将 CANtieCAR 接口（随捆绑包提供）连接到计算机上的 USB 端口！

许可证转移功能

如果您需要更改或重新安装您的计算机，您可以使用 Multiecuscan 的许可证转移功能将许可证转移到新系统。

以下是如何使用许可证转移功能：

  1. 在您有许可证的计算机上打开 Multiecuscan。

  2. 转到“注册”窗口并单击“停用”按钮

  3. 系统会生成一个Removal Key（见下图）

  4. 记下注册窗口中的所有 3 个密钥——硬件密钥、许可证密钥、删除密钥。

  5. 在新电脑上安装Multiecuscan。

  6. 打开注册窗口并记下新的硬件密钥。

  7. 转到 http://www.multiecuscan.net/TransferLicense.aspx 并输入旧数据（硬件密钥、许可证密钥、删除密钥）和新硬件密钥以及您的电子邮件，然后单击“请求许可证”。

  8. 您将收到一封包含新许可证密钥的电子邮件。

如果您对许可证转移功能有任何疑问，可以将新的硬件密钥通过电子邮件发送至 support@multiecuscan.net。

请注意，许可证转移功能对转移次数有限制。

2. 用户界面

2.1. Multiecuscan 主屏幕

当您启动 Multiecuscan 时，它会显示以下屏幕：

Multiecuscan 的主屏幕分为 4 个部分。 您可以使用 TAB 按钮或鼠标从一个部分转到另一个部分。 您还可以使用箭头键在列表中上移/下移。

  1. 在这里您可以选择您车辆的“品牌”。

您还有一个额外的选择选项最近，您可以在其中查看您最近使用过的 20 款车型。 您可以从“设置”中清除此列表。

  1. 在这里您可以选择车辆的型号和版本。

您还可以从键盘输入字母以进行快速搜索

您还可以从键盘键入字母以快速搜索此列表。例如，您可以从键盘键入“STI”，列表将滚动到 STILO。

  1. 在这里您必须选择您要连接的系统类型。

  2. 此处系统将显示所选车辆品牌、型号、版本和系统类型支持的控制模块。 您必须选择汽车中的控制模块。 如果您选择了不正确的模块并连接到它，程序将警告您消息“不正确的 ISO 代码”。

要查看您的汽车中可用的模块，您可以使用 Multiecuscan 的 SCAN 功能（见下文）。

  1. 单击此按钮（或按键盘上的 F10 按钮）开始与所选模块的连接。 为了连接到模块，您必须确保：
    • 接口驱动程序安装正确并且接口的“延迟”设置设置为最小
    • 界面在 Multiecuscan 设置中正确配置（请参阅下面的设置屏幕）
    • 接口连接到 USB 端口。 请注意，大多数接口始终需要连接到计算机的同一个USB端口，否则COM端口号会发生变化，您需要在Multiecuscan设置中更改接口配置
    • 接口连接到车辆。
    • 连接了正确的接口适配器（如果需要）。 当您单击“连接”按钮时，如果需要适配器，程序将警告您。 如果您不想看到所需适配器的警告消息，则可以从 Multiecuscan 设置中将其关闭。
    • 车辆钥匙已打开（位置 MAR）。

“模拟”模式可让您探索程序的功能，而无需实际连接到汽车。 使用此模式，您可以查看哪些参数、测试和校准可用于任何控制模块。

请注意，系统只会发现可以使用连接的接口适配器访问的模块（例如：如果连接了 RED 接口适配器，则系统只会发现安全气囊控制模块），但是，如果您使用的是 MULTIPLEXED 带有 CANtieCAR 接口的版本，那么系统将发现您车辆中的所有模块。

  • 接口驱动程序安装正确并且接口的“延迟”设置设置为最小
  • 界面在 Multiecuscan 设置中正确配置（请参阅下面的设置屏幕）
  • 接口连接到 USB 端口。 请注意，大多数接口始终需要连接到计算机的同一个USB端口，否则COM端口号会发生变化，您需要在Multiecuscan设置中更改接口配置
  • 接口连接到车辆。
  • 连接了正确的接口适配器（如果需要）。
  • 车辆钥匙已打开（位置 MAR）。

2.2. Multiecuscan 设置屏幕

当您单击 Multiecuscan 主屏幕上的“设置”按钮（或按键盘上的 F9）时，程序将打开 Multiecuscan 设置窗口。 设置窗口分为 3 个部分——常规、接口和图形。 有关每个选项的详细说明，请参见下文。

  1. 在这里您可以选择 Multiecuscan 用户界面的语言。 这是程序显示的所有按钮、列表、帮助文本和各种消息的语言。

输入值后，您可以单击“确定”按钮（或按键盘上的 Enter 键）继续该过程。 您也可以单击取消按钮（或按键盘上的 ESC 键）中止执行。

3. 连接到您的汽车（示例）

3.1. 扫描功能

SCAN 功能允许您扫描您的汽车以查找控制模块。 单击 Multiecuscan 主屏幕上的“扫描”按钮（或按键盘上的 F11）开始扫描您的车辆。 该程序将开始扫描您的车辆并显示发现的控制模块的结果。

请注意，系统只会发现可以使用连接的接口适配器访问的模块（例如：如果连接了 RED 接口适配器，则系统只会发现安全气囊控制模块），但是，如果您使用的是 Multiecuscan 带有 CANtieCAR 接口的 MULTIPLEXED 版本，然后系统将发现您车辆中的所有模块。

另请注意，为了成功扫描，您必须确保： - 接口驱动程序已正确安装和配置

  • 接口在 Multiecuscan 设置中正确配置
  • 接口连接到USB端口
  • 接口连接到车辆。
  • 连接了正确的接口适配器（如果需要）。
  • 车辆钥匙已打开（位置 MAR）。

结果包含有关每个已发现模块的基本信息——类型、描述、ISO 代码。 ISO 代码是每个控制模块修改的唯一标识符。 结果如下所示：

3.3. 连接到发动机控制模块

在这个例子中，我们将连接到 Fiat Stilo 1.9 JTD 的引擎控制模块，然后检查错误并检查一些实时数据。

[第 1 步：] 将接口连接到计算机和车辆的 OBD 端口。

[第 2 步：] 选择品牌、型号、版本和控制模块。

[第 3 步：] 单击“连接”按钮（或按键盘上的 F10）。 Multiecuscan 将开始连接到您的汽车。

[步骤 4:] 成功连接后，Multiecuscan 将显示信息屏幕。

[第 5 步：] 现在我们可以进入错误屏幕并检查存储的故障代码。

如您所见，该控制模块报告了 2 个错误。 实际上，我在引擎工作时通过拔下 MAF 连接器故意制造了这两个错误。

在左下方的窗口中，您可以查看有关检测到错误代码时的情况的详细信息。

不管怎样，既然我们知道我们已经修复了错误的原因（我把连接器插回去），我们就可以清除错误。

只需单击“清除错误”按钮（或按键盘上的 F10），Multiecuscan 就会清除这些错误。

[第 6 步：] 现在我们可以进入参数屏幕并检查一些实时数据。

这里我们选择了 3 个参数——发动机转速、总燃油量和喷射时间。

Multiecuscan 将所选参数标记为红色，并开始显示这 3 个参数的实时数据。

[第 7 步：] 现在我们可以转到“图形”屏幕并对实时数据进行 CSV 记录。

在这个屏幕上，我们可以看到我们已经在参数屏幕上选择的参数。

屏幕左侧显示我们参数的实时数据。 它还在数据记录期间监控最小/最大值。

您可以单击“开始/停止”按钮来创建录音。

[第 8 步：] 现在我们可以转到执行器屏幕并进行一些测试。

我们看到了可用于此控制模块的所有测试。 我们将选择“Fan 1st speed”并执行它。

我们按 F10，Multiecuscan 将开始执行此测试。 发动机冷却风扇开始低速工作几秒钟。

测试完成后，Multiecuscan 将显示消息“已完成”。

[第 9 步：] 断开与发动机控制模块的连接。 您可以随时通过按 F11 断开与控制模块的连接并返回到 Multiecuscan。

3.4. 服务间隔重置

在这个例子中，我们将连接到菲亚特 Stilo 的仪表盘控制模块，并重置服务距离。 换油后通常需要进行此操作。

[第 1 步：] 将接口连接到计算机和车辆的 OBD 端口。

[第 2 步：] 选择品牌、型号、版本和控制模块。

然后单击“连接”按钮（或按键盘上的 F10）。 Multiecuscan 将开始连接到您的汽车。

[步骤 3:] 成功连接后，Multiecuscan 将显示信息屏幕。

[第 4 步：] 现在我们可以进入参数屏幕并检查当前的“服务距离”和“距离下一次服务的天数”。

正如我们在下面的屏幕截图中看到的那样，最后一次“服务重置”是在 113,104 公里处执行的，现在里程表是 124,219 公里。 此外，最后一次“服务重置”是在 15/06/2011 执行的。 我们还可以看到距离汽车必须进行维修还有 1245 公里或 126 天。

我们还可以看到还剩下 7 张“服务优惠券”（当此值为 0 时，仪表板上的服务到期消息将被禁用）。

[第 4 步：] 现在我们可以进入“调整”屏幕并执行服务重置程序。

正如您在下面的屏幕上看到的，也可以运行其他程序（例如直接设置下一次服务的日期和距离），但“服务重置”程序是推荐的方法。 此外，不能保证其他程序适用于每辆车。

[第 5 步：] 现在我们选择“NEXT SERVICE KM RESER”并按 F10。 然后系统会要求我们确认我们已经阅读了所有笔记。

[步骤 6:] 系统会开始执行程序，然后显示结果。

[第 7 步：] 现在我们可以转到参数并检查更新的值…

[第 8 步：] 断开与控制模块的连接。 您可以通过按 F11 断开与控制模块的连接并返回到 Multiecuscan 主屏幕。

请注意，对于配备 DPF 的车辆，您还需要重置发动机控制模块中的换油指示器。 您必须连接到发动机控制模块，然后转到调整并执行“换油”程序。

3.5. 连接到引擎并更改喷油器 IMA 代码

在此示例中，我们将连接到 Fiat Grande Punto 1.3 Multijet 的发动机控制模块，并将更改喷油器的 IMA 代码。

[第 1 步：] 将接口连接到计算机和车辆的 OBD 端口。

[第 2 步：] 选择品牌、型号、版本和控制模块。

[第 3 步：] 单击“连接”按钮（或按键盘上的 F10）。 Multiecuscan 将开始连接到您的汽车。

[步骤 4:] 成功连接后，Multiecuscan 将显示信息屏幕。

[第 5 步：] 现在我们按 F7 进入“调整”屏幕。

[第 6 步：] 现在我们选择“Injector code Cyl. 3”并按 F10。 然后系统会要求我们确认我们已经阅读了所有笔记。

[第 7 步：] 我们通过按 Y 确认我们已经阅读了注释。然后系统将显示一个数据输入窗口，其中包含当前 IMA 代码（从发动机控制模块获得）。

[第 8 步：] 现在我们可以输入新的注入器代码。 我们只需从键盘键入 bv742dv5u（或者我们可以使用鼠标并更改每个字符）。

然后我们按“Enter”键进入下一步（或者我们点击OK按钮）。 我们也可以按取消按钮并中止程序。

**[第8步：]**现在系统会要求您确认您输入的数据是否正确。

您可以按 Y 进入下一步并将新代码写入引擎控制模块。

您也可以按 N 中止该过程。

按 Y 后，系统将开始将新代码写入发动机控制模块。

如果新代码不正确，程序可能会在这里失败。

[第 9 步：] 现在您可以转到参数并检查“Injector code Cyl. 3”的值。

[第 10 步：] 断开与发动机控制模块的连接。 您可以通过按 F11 断开与控制模块的连接并返回到 Multiecuscan 主屏幕。

3.6. 连接到 PROXI Alignment 并激活 Fiat Stilo 上的巡航控制灯

在这个例子中，我们将连接到 Fiat Stilo 并使用 PROXI Alignment 程序设置来启用仪表板上的巡航控制灯。

[第 1 步：] 将接口连接到计算机和车辆的 OBD 端口。

[第 2 步：] 选择品牌、型号、版本和控制模块。

[第 3 步：] 单击“连接”按钮（或按键盘上的 F10）。 Multiecuscan 将开始连接到您的汽车。

成功连接后，Multiecuscan 将显示信息屏幕。

在这个例子中，汽车配置正确，不需要对齐。 因此，系统会警告我们不需要 PROXI Alignment。

我们可以按 Y 继续。

[第 4 步：] 按 F7 进入“调整”屏幕。

[第 5 步：] 选择“巡航控制灯”并按 F10 执行此调整。 系统将显示设置的当前值。

[第 6 步：] 将值更改为“已启用”并按 Enter（或单击“确定”）。

**[第7步：]**系统会确认操作完成。

[第 8 步：] 现在我们必须执行 PROXI 对齐程序，将新设置写入汽车。

所以，我们选择“PROXI ALIGNMENT PROCEDURE”并按F10。 系统会要求您确认是否要继续。

[第 9 步：] 在我们确认要继续后，按 Y Multiecuscan 将启动程序。

当流程完成时，系统将显示状态（成功或失败）。

[第 10 步：] 成功执行后，系统会要求您将汽车钥匙转到 OFF 位置。

就这些。 现在，当您打开巡航控制时，巡航控制灯应该已启用并正常工作。

3.7. 向 Multiecuscan 服务器发送诊断报告

诊断报告帮助我们发现并修复 Multiecuscan 中的问题。 该报告包含有关您上次连接的信息（发送/接收到您的汽车的数据）。 它不包含任何个人信息或有关您的计算机的任何信息。

Multiecuscan 在 Multiecuscan 主窗口的右下角显示一个红色的小按钮“发送报告”。 该按钮在您断开与控制模块的连接或连接失败后出现。

[第 1 步：] 连接到控制模块。

[第 2 步：] 转到错误、参数等。

[第 3 步：] 按 F11 断开与控制模块的连接。

[第 4 步：] 点击红色的“发送报告”按钮。 执行此操作时，您的计算机必须连接到互联网。

系统会要求您确认。 按 Y 确认。
